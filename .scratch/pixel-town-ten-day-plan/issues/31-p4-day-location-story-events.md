# P4：接入白天地点首访教程与剧情事件

Status: done
Milestone: P4
Type: AFK
User stories: 11, 16, 19, 23, 27, 37, 38, 57, 58, 61, 73, 74

## What to build

在 Issue 30 的确定性地点剧情选择器上，将餐馆、便利店和图书馆从“每次进入重复固定初识对话”升级为“首访教程 + 后续日常/意外事件”。内容以 `docs/story/LOCATION_EVENT_SCHEDULE.md` 为契约，继续复用共享对话框、人物 fallback、固定 NPC 热点和原地点玩法。

本切片不改变订单、需求、库存、题库、整理任务、收益或随机规则。意外事件先通过 NPC 对话和当日提示表达，关闭或跳过后仍进入现有玩法。

## Acceptance criteria

- [x] 餐馆、便利店和图书馆在各自完成访问次数为 0 时显示专属说明与操作指引，不依赖全局第 1 天。
- [x] 每个地点完成过一次后，再进入时根据当前游戏日和确定性事件选择器显示日常、意外或第 10 天回收文本，不再默认重复初识台词。
- [x] 每个地点至少接入 1 个首访教程、3 个普通/当日事件、2 个意外事件和 1 个第 10 天回收脚本。
- [x] 第 5 天雨天和第 8 天旧集市碎片在对应地点有明确文本；错过当天不补播、不阻塞玩法。
- [x] 对话 active 时冻结餐馆倒计时、便利店经营输入和图书馆模式/题目输入；继续、跳过和点击不泄漏。
- [x] 对话关闭后只交接一次既有地点流程，不能重复创建订单、经营方案、题目或整理任务。
- [x] 事件文本不直接修改玩家属性、库存、访问次数或行动结果；访问次数仍只由完成结果成功应用时更新。
- [x] 存档恢复后相同地点边界显示同一事件；完成并进入下一次访问后才可能选择下一事件。
- [x] 所有新文本进入字形清单，使用共享 UTF-8 三行/分页布局。
- [x] 无窗口测试分别覆盖三地点的首访、后续事件、意外事件、读档确定性、输入冻结和原玩法交接。
- [x] 诊断截图覆盖三地点首访、至少一个雨天意外、至少一个旧集市事件和第 10 天回收，文字无缺字、重叠或越框。
