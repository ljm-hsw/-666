# P4：接入五地点室内移动、静态碰撞与邻近互动

Status: ready-for-human
Milestone: P4
Type: HITL
User stories: 57, 71, 72, 75

## What to build

在餐馆、便利店、图书馆、酒馆和家五个固定镜头室内场景中，为主角接入 WASD/方向键移动、静态矩形碰撞、门口出生/退出以及邻近 E/Space 互动。鼠标点击 NPC、桌面、床铺和固定按钮继续作为等价路径。

本切片不恢复 NPC 轨迹、巡逻、寻路或动态避障，不把小镇地图改成自由移动场景，不改变地点规则、剧情选择、行动结果、数值、阶段或 v2 存档。主角位置、朝向和移动动画时间只存在于当前场景运行期，重进场景时重置。

## Acceptance criteria

- [x] raylib-free 导航模块只接收显式帧时间、四向轴和互动意图，并返回主角位置、朝向、移动状态及最近互动目标的只读 presentation。
- [x] `WASD` 与方向键提供相同的四向输入；斜向速度归一化，不比单轴移动更快。
- [x] 五个场景都有 960×540 可行走边界、合法出生点、门口退出触发和与背景家具对齐的静态碰撞体；门洞不被墙体或家具封住。
- [x] 24×24 主角脚底碰撞框不能穿过家具或墙体；轴分离移动允许沿障碍边缘滑动。
- [x] 餐馆老板、便利店店主、图书管理员、酒保、五子棋桌、骗子骰子桌和家中床铺支持邻近 E/Space 互动；多个目标重叠时选择距离最近者。
- [x] 邻近互动委托既有对话、桌游、休息和返回入口，不直接修改玩家属性或绕过 `GameSession`/地点 Runtime。
- [x] 对话 active 时移动、互动、返回和地点玩法输入不泄漏；关闭后恢复导航。
- [x] 主角导航在离开大厅、开始玩法、结算或返回地图时关闭；再次进入时从该地点出生点重置，不新增存档字段。
- [x] F3 显示可行走边界、静态碰撞体、门口退出和主角脚底盒；普通模式不显示诊断框。
- [x] 自动化测试覆盖布局合法性、出生点、边界、门洞、碰撞/滑动、斜向等速、最近目标与图书馆/酒馆邻近互动。
- [x] macOS 图形会话成功导出 57/57 张 UI 诊断截图，其中六张覆盖餐馆老板、便利店店主、图书管理员、酒馆双桌和家中床铺；主角、碰撞体和底部提示无顶部遮挡。
- [x] 在 macOS 实际窗口中确认物理 WASD 可让主角离开出生点，靠近固定 NPC 后使用键盘互动可打开既有对话；测试过程可继续完成地点并推进游戏日。
- [ ] 在 macOS 实际窗口中打开 F3，持续朝家具移动确认无法穿过，并沿家具边缘斜向移动确认可滑动；餐馆、便利店、图书馆、酒馆和家各检查一次。
- [ ] 在目标 Windows 机器重复 WASD/方向键与 E/Space 烟雾测试，并确认无平台输入差异。

- 2026-07-16：按 TDD 恢复 `scene_collision` 的运行时能力并新增 `SceneNavigationRuntime`。规则模块不链接 raylib；应用层只把 `IsKeyDown(W/A/S/D/方向键)` 与 `IsKeyPressed(E/Space)` 转为显式输入，再把邻近目标映射回既有对话、桌游、回家或退出流程。
- 2026-07-16：全量 284/284 个 doctest 用例（3070/3070 个断言）与 CTest 1/1 通过；真实 OpenGL/audio/resource 启动成功并导出 57/57 张 UI 诊断截图。macOS Computer Use 能读取窗口并移动悬停指针，但其辅助功能 click/press_key 没有产生 GLFW 可见的低层按键按下事件，因此不能把该自动化尝试写成物理 WASD 通过；本 issue 保持 `ready-for-human`，等待人工键盘验收。
- 2026-07-16：随后由用户实际操作隔离的 `midgame` macOS 测试包。只读屏幕历史确认测试从第 5 天推进至第 7 天，经过餐馆、便利店、酒馆、家和图书馆；便利店与图书馆画面中主角已离开门口出生点，且在鼠标不位于 NPC 热点时分别打开店主/管理员对话。该证据证明物理 WASD 输入、邻近目标和键盘对话交接真实生效；尚未获得可区分“持续撞墙/沿边滑动”的实机记录，因此碰撞手感与 Windows 项仍保留人工门禁。
- 2026-07-16：根据实机画面将室内主角从 36×54 调整为与固定 NPC 一致的 44×66 显示尺寸，脚底中心锚点和 24×24 玩法碰撞框保持不变，避免视觉放大改变可行走范围或互动判定。
