# P4：收口酒馆固定 NPC 待机与点击热点

Status: done
Milestone: P4
Type: AFK
User stories: 29, 30, 32, 54, 57, 58, 71, 72

## What to build

酒馆保留固定镜头与点击交互：酒保固定在吧台循环播放现有六帧待机图，点击酒保打开共享短对话；玩家点击五子棋桌、骗子骰子桌或固定玩法按钮进入既有挑战选择与真实桌游流程。

本切片不加入主角移动、常客巡逻、路点、寻路或运行时碰撞，不改变 `TavernRuntime`、两种桌游规则、赌注、隐藏信息或 Settlement 契约。

## Acceptance criteria

- [x] 酒保固定在吧台热点，循环播放仓库内六帧待机 sprite；资源缺失时程序 fallback 仍可点击。
- [x] 点击酒保进入 Issue 23 的共享对话；对话 active 时重复点击、返回和挑战输入不泄漏。
- [x] 点击五子棋、骗子骰子热点或玩法按钮进入既有挑战选择和挑战流程。
- [x] 不存在主角移动、常客巡逻、路点、寻路或角色碰撞代码路径。
- [x] `TavernRuntime` 的隐藏信息、终局重试、挑战类型和 Settlement 行动结果契约保持不变。
- [x] 酒保待机动画时间不进入玩家状态、酒馆战绩、行动结果或 v1 存档；重新进入时重置。
- [x] 静态碰撞数据只用于可选 F3 美术诊断，不参与酒保、桌面热点或挑战判断。
- [x] 无窗口测试覆盖酒保热点、共享对话、待机时间、挑战进入、隐藏信息、终局重试和核心状态不变。
- [x] 诊断截图覆盖酒馆大厅、酒保对话、五子棋、骗子骰子和资源 fallback，文字无缺字、重叠或越框。
