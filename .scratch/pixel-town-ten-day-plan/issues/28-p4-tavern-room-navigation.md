# P4：接入酒馆室内移动、静态碰撞与常客轨迹

Status: needs-info
Milestone: P4
Type: AFK
User stories: 29, 30, 32, 54, 57, 58, 71, 72

## What to build

复用图书馆已经验证的固定室内导航边界，接入酒馆房间：玩家使用上下左右方向键在可行走范围内移动，避开吧台、桌椅、家具和墙体，靠近酒保后触发共享短对话，靠近桌面热点后进入现有五子棋或骗子骰子选择与挑战流程。酒保保持岗位待机，一名常客可以按作者配置的确定性路点活动。

本切片不改变 `TavernRuntime`、两种桌游规则、赌注或 Settlement 契约。允许使用程序图形或单帧占位人物；主角和 NPC 场景位置不写入 v1 存档，并在每次进入时重置。

## Acceptance criteria

- [ ] 酒馆复用 Issue 27 的 raylib-free 导航 Interface、碰撞语义和只读 presentation，不复制第二套移动系统。
- [ ] 主角移动只绑定上下左右方向键，不绑定 WASD；E/Space 或鼠标只在邻近有效目标时触发互动。
- [ ] 场景定义可行走边界、出生点、互动半径、吧台/桌椅/家具/墙体静态碰撞体和常客路点。
- [ ] 主角与常客均不能越界或进入静态碰撞体；靠近吧台和桌边时仍能稳定触发目标而不会穿进物件。
- [ ] 酒保原地待机，常客按确定性路点循环、往返或停留；两者不使用 A*、动态避障或跨场景日程。
- [ ] 靠近酒保触发 Issue 23 的共享对话；对话 active 时主角、常客和酒馆计时冻结，输入不泄漏。
- [ ] 靠近桌面热点可进入既有五子棋或骗子骰子选择与挑战；返回大厅时恢复有效室内状态，不重复结算或额外消耗夜晚行动。
- [ ] `TavernRuntime` 的隐藏信息、终局重试、挑战类型和 Settlement 行动结果契约保持不变。
- [ ] 场景角色位置、朝向和路点进度不进入玩家状态、酒馆战绩、行动结果或 v1 存档；重新进入时从确定起点重置。
- [ ] 新增、移动、缩放或替换吧台、桌椅和家具时必须同步静态碰撞体，并通过视觉与碰撞一致性检查。
- [ ] 无窗口测试覆盖边界、障碍、确定性轨迹、邻近互动、对话冻结、进入/返回挑战、场景重入和核心状态不变。
- [ ] 诊断截图覆盖出生点、吧台碰撞、酒保对话、常客轨迹和桌面挑战入口，文字无缺字、重叠或越框。
