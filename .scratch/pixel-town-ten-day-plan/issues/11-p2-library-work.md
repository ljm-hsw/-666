# P2：接入数据驱动的图书馆工作

Status: done
Milestone: P2
Type: AFK
User stories: 23-27, 54, 61, 62

## What to build

用真实图书馆分类匹配替换模拟白天结果。读者提出文本需求，玩家从有限书籍类别中选择答案，立即看到正确或错误反馈，完成一轮后获得知识、声望及其他已配置变化。

类别和需求由仓库数据内容提供，规则引擎负责匹配和统计，界面不得拥有另一套硬编码答案。

## Acceptance criteria

- [x] 白天阶段可以从地图进入图书馆，非白天阶段不可进入。
- [x] 首次进入显示可跳过、可重新查看的中文分类说明。
- [x] 书籍类别和读者需求从数据内容加载，缺失或非法数据进入明确错误路径。
- [x] 每条需求映射到一个正确类别，玩家选择后立即获得一致反馈。
- [x] 一轮结束汇总正确数和表现，只生成一个统一行动结果。
- [x] 一轮结束返回未修正的基础行动结果，核心系统统一应用知识、声望、体力、心情、已配置金钱变化和集中配置的心情修正。
- [x] 图书馆行动摘要遵循 `docs/story/STYLE_GUIDE.md`，通过剧情文本入口生成借书卡、旧地图、泛黄书页等图书馆回声，不在 UI 绘制代码中硬编码长文案。
- [x] 固定种子复现需求顺序，后期难度通过内容和集中参数调整而不改变基础规则。
- [x] 规则测试覆盖正确匹配、错误匹配、非法内容和奖励汇总。
- [x] 集成测试覆盖地图进入、完成、结果展示和返回夜晚选择。
