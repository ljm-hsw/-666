# P4：新增图书馆书籍整理模式

Status: done
Milestone: P4
Type: AFK
User stories: 23, 27, 54, 61, 62

## What to build

在保留 P2“读者咨询/类别答题”完整闭环的前提下，为图书馆新增可独立选择的书籍整理模式。玩家从场景中的散落书或错架书开始，拿起一本书后点击正确分类书架完成归位；错误分类必须提供即时反馈并允许原地重试。

整理规则、内容数据、raylib 展示和核心行动结算保持分层。后续 Issue 27 的管理员邻近互动只负责进入共享模式选择，不复制或改写两种工作规则。

## Acceptance criteria

- [x] 图书馆开始行动后先显示“读者咨询 / 书籍整理”模式选择，原有咨询模式不回归。
- [x] 整理规则是 raylib-free 独立模块，只通过公开接口暴露任务、持书、进度、反馈和结算。
- [x] 书架与整理任务来自 `assets/data/library_data.txt`，非法类别、错架来源或重复 ID 进入明确错误路径。
- [x] 散落书和错架书都可拾取；拿着一本书时不能再拿第二本。
- [x] 放到错误书架后书不会丢失，仍保持持书状态并可改放正确书架。
- [x] 固定日种子复现任务顺序，任务数量由集中配置限制。
- [x] 每个完成任务只奖励一次；散落来源和错架来源不重复计奖，主动放弃不结算部分奖励。
- [x] 整理结果使用当前 `active_result_id` 映射为图书馆统一 `ActionResult`，完成后进入夜晚选择。
- [x] 模式选择和整理页面使用 960×540 逻辑画布、60px 独立顶栏和当前图书馆背景；顶栏不覆盖窗户。
- [x] UI 明确显示进度、错误次数、持书书名、操作提示、错误反馈和返回入口，不依赖 emoji 字形。
- [x] 无窗口测试覆盖正确归位、错误重试、确定性、数据解析、单次奖励、放弃和核心结算。
- [x] 诊断截图覆盖模式选择和错误后持书重试状态；整理初始状态由无窗口规则测试覆盖。
