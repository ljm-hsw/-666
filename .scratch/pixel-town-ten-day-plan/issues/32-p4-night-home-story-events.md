# P4：接入酒馆与家首访教程和剧情事件

Status: done
Milestone: P4
Type: AFK
User stories: 28, 29, 34, 37, 38, 57, 58, 61, 73, 74

## What to build

在 Issue 30 的确定性地点剧情选择器上，将酒馆和家从固定酒保初识/固定回家独白升级为“首访教程 + 后续日常/意外事件”。酒馆继续使用固定酒保和桌面热点；家继续使用完整背景与确认休息流程。内容以 `docs/story/LOCATION_EVENT_SCHEDULE.md` 为契约。

本切片不改变五子棋、骗子骰子、赌注、酒馆战绩、休息收益或每日总结规则。意外事件只改变对话和提示，关闭或跳过后才交回原流程。

## Acceptance criteria

- [x] 酒馆首次有效访问说明玩法选择、赌注和一次性结算；后续访问根据游戏日选择常客、棋子、照片或第 10 天回收事件。
- [x] 家首次确认休息前由镇长访客说明夜晚消耗和恢复语义；后续每次回家确认休息均按游戏日选择餐馆老板、便利店店主、管理员或镇长来访，第 10 天由镇长回收。
- [x] 酒馆和家各至少接入 1 个首访教程、3 个普通/当日事件、2 个意外事件和 1 个第 10 天回收脚本。
- [x] 酒馆事件关闭后仍由现有桌面热点进入挑战，不自动下注、不提前开始游戏；酒保对话和桌游输入不泄漏。
- [x] 家事件关闭或跳过后只提交一次既有 `home_rest_result()`；重复输入不能重复恢复、推进阶段或写入总结。
- [x] 事件文本不修改玩家属性、酒馆战绩、访问次数或行动结果；访问次数只在相应完成结果成功应用时更新。
- [x] 存档恢复后重播同一个无奖励事件，不保存当前行号；同一边界不会重新抽取另一事件。
- [x] 所有新文本进入字形清单，使用共享 UTF-8 三行/分页布局。
- [x] 无窗口测试覆盖酒馆/家首访、后续事件、意外事件、读档确定性、桌游交接和休息只应用一次。
- [x] 诊断截图覆盖酒馆首访、黑棋子/旧照片事件、家首访、雨夜独白和第 10 天回收，文字无缺字、重叠或越框。
