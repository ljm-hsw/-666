# P2：餐馆背景、食物与时间表现视觉规划

Status: done
Milestone: P2
Type: AFK
User stories: 11-15, 35, 54, 61, 62

## What to build

为餐馆玩法定义可落地的 960×540 视觉布局和资源槽位，使后续可以接入 imagegen 餐馆室内背景、食物图标、订单票据、倒计时条和反馈标记，同时不阻塞当前程序绘制 UI 和规则测试。

本 issue 的目标是把“餐馆需要背景 UI、各种食物、对应时间和说明可视化展示”写成团队可执行规格。若本阶段没有最终素材，可以先使用程序绘制占位和 Kenney/imagegen 候选资源；最终视觉验收仍归 P4。

## Acceptance criteria

- [x] 定义餐馆页面 960×540 布局：状态栏、背景安全区、订单区、食物选择区、时间区、反馈区、统计/结算区。
- [x] 规划餐馆室内背景资源：吧台、餐桌、厨房/上菜窗口、暖色像素风，不包含任何生成文字。
- [x] 规划五个食物图标资源：炒饭、面条、汤、饺子、沙拉；小尺寸下仍能区分。
- [x] 食物图标缺失时有程序绘制或文字按钮 fallback，不影响规则测试和流程。
- [x] 时间表现必须同时有数字和可视化条形/颜色变化，不只显示文本秒数。
- [x] 订单票据和反馈标记不把可读中文烘焙进图片，所有文字由程序字体绘制。
- [x] 说明卡、订单票据、倒计时、食物按钮和结算页使用统一面板/按钮风格，避免每个状态单独一套样式。
- [x] 资产进入仓库前必须记录来源、用途、候选/最终状态、hash 和许可/生成说明。
- [x] 文档同步到 `docs/IMAGEGEN_VISUAL_PLAN.md` 或 `docs/UI_ASSET_STRATEGY.md`，便于 P4 统一收口。
