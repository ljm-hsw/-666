# P2：接入跨日库存的便利店经营

Status: ready-for-agent
Milestone: P2
Type: AFK
User stories: 16-22, 37, 38, 54, 61, 62

## What to build

用真实便利店经营替换模拟白天结果。玩家读取天气和事件提示，在现有现金与库存基础上选择进货数量和低、标准、高价格档位，执行一次可复现销售模拟，查看成本、销量、收入、利润和剩余库存。未售商品进入下一游戏日并写入存档。

## Acceptance criteria

- [ ] 白天阶段可以从地图进入便利店，非白天阶段不可进入。
- [ ] 首次进入显示可跳过、可重新查看的进货、定价和需求说明。
- [ ] 进货不能使现金低于合法下限，价格只能选择三个已定义档位。
- [ ] 经营前展示天气和事件提示，但不展示准确销量。
- [ ] 需求模型组合基础需求、价格档位、天气/事件修正和固定种子波动。
- [ ] 相同状态、配置和种子产生相同销量，不同合法决策可以产生不同结果。
- [ ] 结算展示成本、销量、收入、利润和剩余库存，并只生成一个未修正的基础行动结果；核心系统统一应用集中配置的心情修正。
- [ ] 未售库存进入下一游戏日，保存恢复后商品和数量保持一致。
- [ ] MVP 不产生保质期、自然损耗或仓储容量行为。
- [ ] 规则测试覆盖预算、价格档位、需求修正、库存守恒和跨日存档。
- [ ] 集成测试覆盖地图进入、经营、结果应用、下一日库存和返回夜晚选择。
