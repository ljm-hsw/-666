# P2：接入完整餐馆白天工作

Status: ready-for-agent
Milestone: P2
Type: AFK
User stories: 11-15, 35, 54, 61, 62

## What to build

用真实餐馆玩法替换模拟白天结果。玩家从地图进入餐馆，查看顾客订单，通过点击菜品并提交上菜，在有限等待时间内完成服务，最后查看服务表现和统一行动结果并返回夜晚选择。

餐馆规则必须独立于渲染，错单、超时和正确服务先形成地点内部统计，再在结束时生成一次行动结果。

## Acceptance criteria

- [ ] 白天阶段可以从地图进入餐馆，非白天阶段不可进入且给出原因。
- [ ] 首次进入显示可跳过、可重新查看的中文玩法说明。
- [ ] 顾客订单、等待状态、菜品选择和提交反馈在 UI 中清晰可见。
- [ ] 正确订单、错误订单和顾客超时遵循 PRD 规则并进入服务统计。
- [ ] 暂停、失焦和最小化期间订单计时冻结。
- [ ] 模块结束只生成一个未修正的基础行动结果，核心系统统一应用金钱、体力、声望、心情变化和集中配置的心情修正。
- [ ] 主动放弃消耗白天阶段且无收益；异常关闭按阶段边界恢复。
- [ ] 固定种子可以复现订单序列，难度只通过集中数值参数增长。
- [ ] 无窗口规则测试覆盖正确订单、错单、超时、统计和结果计算。
- [ ] 集成测试覆盖地图进入、完成、结果展示和返回夜晚选择。
