# P2：餐馆 UI、提示与操作反馈收口

Status: done
Milestone: P2
Type: AFK
User stories: 11-15, 35, 54, 61, 62

## What to build

在不改变餐馆规则引擎和核心状态机的前提下，把当前餐馆从“基础可运行 UI”收口为“玩家能看懂、能操作、能感知反馈”的白天工作场景。

本 issue 负责教程、订单展示、食物按钮、倒计时、正确/错单/超时反馈和结算页信息结构。最终美术资源不作为完成前提，但布局必须为后续餐馆背景、食物图标和 UI 装饰替换预留位置。

## Acceptance criteria

- [x] 首次进入餐馆显示中文玩法说明卡，包含目标、操作、收益、风险和结束方式。
- [x] 说明卡明确提示 `1-5` 菜品快捷键、鼠标点击菜品、等待归零会超时、错单影响表现、完成后需要结算。
- [x] 当前顾客订单以“订单票据”或等价视觉区域展示，包含菜品、当前第几单、总单数和剩余时间。
- [x] 五个菜品按钮在 960×540 下清晰可读，并同时支持鼠标点击与键盘 `1-5`。
- [x] 倒计时以数字和进度条/颜色变化同时展示，剩余时间低时有明显视觉警示。
- [x] 正确上菜、错单和超时都有短反馈，不只依赖最终统计。
- [x] 主动放弃入口清晰，文案说明会消耗白天阶段且无收益。
- [x] 完成页展示正确数、错单数、超时数、正确率、金钱/声望/心情/体力变化。
- [x] 暂停、失焦和最小化期间餐馆倒计时不推进。
- [x] 不把餐馆长剧情摘要硬编码在绘制代码中；最终摘要仍通过后续剧情文本入口收口。
- [x] 自动化测试或可复查截图覆盖：进入说明、订单中、反馈中、完成结算四个状态。
