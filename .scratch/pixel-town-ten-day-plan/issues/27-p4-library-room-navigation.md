# P4：接入图书馆室内移动、静态碰撞与管理员轨迹

Status: ready-for-agent
Milestone: P4
Type: AFK
User stories: 23, 27, 54, 57, 58, 71, 72

## What to build

以图书馆为第一条固定室内导航垂直切片：玩家进入固定镜头房间后，使用上下左右方向键在可行走范围内移动，避开书架、家具、柜台和墙体，靠近管理员后使用 E/Space 或鼠标触发共享短对话，再进入现有“读者咨询 / 书籍整理”模式选择。管理员按作者配置的确定性路点活动，并与主角遵守同一组静态碰撞边界。

本切片允许使用程序图形或单帧占位人物，不改变图书馆题库、奖励、统一行动结果、阶段或 v1 存档。主角和管理员的位置、朝向及路点进度在每次进入场景时从确定起点重置。

## Acceptance criteria

- [ ] 固定室内导航运行期不依赖 raylib，接收显式帧时间、方向和互动输入，并提供只读 presentation。
- [ ] 主角移动只绑定上下左右方向键，不绑定 WASD；同时按下两个方向时速度归一化且不受帧率明显影响。
- [ ] 场景定义明确的可行走边界、主角/管理员出生点、互动半径、静态碰撞体和管理员路点。
- [ ] 主角和管理员均不能越界，也不能进入书架、家具、柜台或墙体范围；轴向贴墙移动不会卡入或穿过障碍。
- [ ] 管理员按确定性路点循环、往返或停留，相同场景定义与输入产生相同轨迹，不实现 A* 或动态避障。
- [ ] 玩家靠近管理员后可使用 E/Space 或鼠标触发共享对话；超出互动半径时显示明确靠近提示且不启动答题。
- [ ] 对话 active 时主角、管理员和图书馆计时冻结，输入不会泄漏；关闭后进入共享模式选择，再由玩家选择读者咨询或书籍整理。
- [ ] 题库、整理任务、剧情事件、反馈、结算、返回地图和主动放弃行为保持可完成，常规管理员文案使用共享 UTF-8 安全对话框。
- [ ] 场景角色位置、朝向和路点进度不进入玩家状态、行动结果或 v1 存档；重新进入时从确定起点重置。
- [ ] 新增、移动、缩放或替换书架和家具时必须在同一变更中同步静态碰撞体，并通过视觉与碰撞一致性检查。
- [ ] 无窗口测试覆盖边界、斜向归一化、各类障碍碰撞、确定性路点、互动距离、对话冻结、场景重入和状态不变。
- [ ] 诊断截图覆盖出生点、贴近书架、靠近管理员、对话 active 和进入答题五个状态，文字无缺字、重叠或越框。
