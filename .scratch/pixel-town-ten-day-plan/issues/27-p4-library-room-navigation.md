# P4：接入图书馆固定管理员待机与点击交互

Status: done
Milestone: P4
Type: AFK
User stories: 23, 27, 54, 57, 58, 71, 72

## What to build

图书馆使用固定镜头和固定管理员热点。玩家从地图进入后先看到完整背景，管理员固定在柜台位置循环播放待机动画；点击管理员或按 Space 打开共享短对话，对话结束后进入现有“读者咨询 / 书籍整理”模式选择。

本切片不提供主角移动、NPC 巡逻、路点、寻路或运行时碰撞，也不改变图书馆题库、整理任务、奖励、统一行动结果、阶段或 v1 存档。正式管理员 sprite 未批准前允许使用程序待机 fallback。

## Acceptance criteria

- [x] 固定图书馆场景运行期不依赖 raylib，只接收帧时间、管理员点击、返回和对话输入，并提供只读 presentation。
- [x] 管理员热点固定在柜台区域，NPC 不移动、不巡逻、不使用路点或碰撞。
- [x] 管理员待机动画时间稳定循环；场景重新打开时从首帧重置，不进入玩家状态、行动结果或 v1 存档。
- [x] 点击管理员或按 Space 触发共享 UTF-8 对话；点击场景其他位置不会启动工作。
- [x] 对话 active 时重复 NPC 点击、返回和玩法输入不会泄漏；关闭或跳过后进入共享工作模式选择。
- [x] 返回地图不会消耗白天行动；题库、整理任务、剧情事件、反馈、结算和主动放弃保持可完成。
- [x] 书籍整理继续使用书本拾取热区和分类书架归位热区，不再预留主角靠近拾取或放回。
- [x] 静态碰撞数据只用于可选 F3 美术诊断，不参与图书馆点击、对话、模式选择或结算。
- [x] 无窗口测试覆盖待机时间、热点激活、模态输入、返回不消耗、工作交接、场景重开和核心状态不变。
- [x] 诊断截图覆盖固定管理员、不同待机帧、对话 active、模式选择和进入答题，文字无缺字、重叠或越框。
