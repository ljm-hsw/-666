# P4：完成内容扩充与数值平衡

Status: ready-for-human
Milestone: P4
Type: HITL
User stories: 37, 38, 62

## What to build

在全部地点和结局闭环稳定后，补充足以支持十日体验的订单、商品、读者需求、天气和事件，并通过固定种子模拟与人工试玩确定最终数值平衡参数。目标是让赚钱、声望、知识、娱乐和均衡路线都可理解、可完成且不存在明显唯一最优策略。

## Acceptance criteria

- [ ] 内容集合覆盖所有已确认地点规则，不加入 PRD 范围外的新系统。
- [ ] 剧情内容扩充遵循 `docs/story/LOCATIONS_AND_NPCS.md`，只在核心玩法稳定后加入 NPC 三段式、旧集市碎片或条件回声。
- [ ] 所有初始状态、边界、消耗、恢复、收益、价格、需求、概率、计时、赌注、清算和结局阈值集中配置。
- [ ] 固定种子批量模拟能够复现，并生成各路线最终状态和结局分布摘要。
- [ ] 不存在因合法选择导致无法继续白天或夜晚流程的软锁。
- [ ] 餐馆、便利店、图书馆、回家和酒馆具有可区分的机会成本与收益倾向。
- [ ] 七种主结局均能通过合理路线达到，普通与失败结局不会因阈值错误吞掉其他结局。
- [ ] 后期压力通过参数或内容提高，不改变地点基础规则或引入动态难度。
- [ ] 人工试玩记录包含单局时长、地点时长、理解困难、显著占优策略和调整结论。
- [ ] 人工试玩记录包含文案长度、阅读负担、剧情理解偏差和需要压缩的页面。
- [ ] 最终参数由人工明确批准，并在本 issue 的 `Comments` 中记录批准基线。
- [ ] 参数固化后完整自动化测试继续通过。
