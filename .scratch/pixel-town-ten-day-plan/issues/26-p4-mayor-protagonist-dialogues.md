# P4：接入镇长与主角生命周期对话

Status: needs-info
Milestone: P4
Type: AFK
User stories: 1, 7, 9, 10, 28, 42, 57, 58

## What to build

使用共享对话系统接入新游戏时镇长与主角的十日约定，以及选择回家休息时的主角短独白。新游戏对话结束后进入第一日地图；回家独白与既有休息结果、每日总结和保存边界组合成一个可解释流程。第十日评议会和主结局继续使用现有结局文本入口，不新增阶段或可领取奖励。

镇长和主角先使用 32×32 占位小人和统一 fallback；最终角色资产、表情和对话框装饰留给 Issue 17 人工批准。

## Acceptance criteria

- [ ] 新游戏确认后展示镇长与主角的 3-4 句线性对话，继续或跳过后只进入一次第一日地图。
- [ ] 开场对话不绕过已有存档覆盖确认，不会提前创建重复行动或重复保存。
- [ ] 夜晚选择回家时先展示 1-2 句主角独白，完成或跳过后只提交一次既有回家结果，并保持体力、心情、夜晚槽和每日总结语义。
- [ ] 对话打开时优先消费鼠标、Enter/Space 和 Esc，不同时触发地图地点或总结按钮。
- [ ] 对话前后除原本的新游戏创建或回家行动外，不产生额外属性、阶段、库存或战绩变化。
- [ ] 第十日评议会和唯一主结局仍由现有结局流程生成，不新增 `GamePhase`。
- [ ] 首版不保存当前对话行号或已读标记；阶段边界恢复后允许重播无奖励对话，且结果确定。
- [ ] 镇长、主角和回家独白文本遵循温暖、日常、不评判的剧情风格。
- [ ] 人物素材缺失时显示统一 fallback；所有台词进入字形清单并满足三行/分页限制。
- [ ] 自动化测试覆盖新游戏到地图、回家到每日总结、存档恢复和第十日结局未回归。
- [ ] 诊断截图覆盖镇长/主角交替说话、回家独白和 fallback。
