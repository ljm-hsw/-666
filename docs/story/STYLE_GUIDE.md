# 文案风格规范

本文定义《像素小镇：十日经营计划》的文案写作边界。它约束开场、地点说明、地点结果、每日总结、结局页、教程提示和后续 NPC 文本。

## 总原则

- 先写场景，再写数值。
- 用具体物件和动作表达情绪。
- 文本短，不解释过度。
- NPC 像普通人说话，不像系统旁白。
- 不把玩家路线写成好坏评价。
- 失败也只是一种生活画像，不做羞辱。

## 推荐写法

| 方向 | 说明 |
| --- | --- |
| 具体物件 | 地图、围裙、雨伞、借书卡、棋子、窗灯、账本、旧海报。 |
| 生活动作 | 端汤、补货、翻书页、折地图、点账本、烧热水、落棋子。 |
| 短句 | 每段只服务一个情绪或信息点。 |
| 轻回应 | 用“有人记得”“桌边抬头”“账本边角多了小勾”表达反馈。 |
| 数值后置 | 数值是反馈，不是文案主体。 |

## 避免写法

| 禁区 | 原因 |
| --- | --- |
| 宏大口号 | 会把轻量小镇日常推成拯救叙事。 |
| 直白说教 | “你应该”“生活的意义是”会替玩家下结论。 |
| 过度煽情 | 项目基调是轻、暖、日常，不是大悲大喜。 |
| 好坏标签 | 赚钱、娱乐、休息、失败都不应被写成道德优劣。 |
| NPC 标签化 | 避免“热心老板”“神秘管理员”这类扁平称呼主导文本。 |

## 示例对照

| 场景 | 避免写法 | 推荐写法 |
| --- | --- | --- |
| 开场 | 你不需要拯救这里，也不需要证明自己。 | 镇长把地图递给你，纸角有点卷。“先住十天吧。小镇不大，走几遍就熟了。” |
| 餐馆 | 你通过努力让餐馆重新充满希望。 | 午饭刚过，老板把围裙往柜台上一搭。“今天汤没洒，已经很不错了。” |
| 便利店 | 你满足了居民的核心生活需求。 | 雨伞卖得比汽水快。店主在账本边角画了个小勾，说晚上应该还能再来两个人。 |
| 图书馆 | 你发现了小镇过去的意义。 | 借书卡盒子有点卡。管理员敲了两下，掉出一张旧集市地图。 |
| 酒馆 | 你的胜利让大家尊敬你。 | 骰盅停下时，旁边的人先笑了。“这局算你运气好。下一把别走。” |
| 失败结局 | 你经营失败，没有完成小镇期待。 | 账本最后一页有几处空白。镇长没有多说，只把地图折好，放回你手边。 |

## UI 文案约束

- 960×540 画布下，正文优先 1-3 行；长文进入剧情卡片或分页。
- 地图页提示应短于结局页文案，避免遮挡地点信息。
- 教程和操作提示使用功能性短句，不混入长剧情。
- 每个页面只突出一个主文本目标：标题页讲进入小镇，地图页讲当天，地点页讲行动，结算页讲回声，结局页讲被如何记住。

## 后续审查清单

- 文案是否围绕“十天之后，小镇会怎样记住你？”
- 是否存在强行拯救小镇、证明自己、完成使命的语气？
- 是否把玩家路线分成好坏？
- 是否能被 UI 清楚显示？
- 是否纳入字体字形覆盖？
- 是否和当前状态机、地点契约、结局判定一致？
