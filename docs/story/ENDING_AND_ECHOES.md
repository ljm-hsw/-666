# 结局与回声设计

本文定义“选择 -> 回声”和七个主结局的叙事职责，用于 P3 正式结局判定和 P4 内容润色。

## 回声原则

回声不是奖励弹窗，也不是额外成就。它是小镇对玩家十天生活方式的回应。

回声应满足：

- 短，具体，可显示。
- 基于玩家真实选择或玩法统计。
- 不改变主流程和数值结算。
- 不评价玩家对错。
- 不与“每局一个主结局”冲突。

## 可用统计来源

| 来源 | 用途 |
| --- | --- |
| 地点访问次数 | 判断餐馆、便利店、图书馆、酒馆、回家路线倾向。 |
| 玩家最终属性 | 辅助解释金钱、声望、知识、心情路线。 |
| 便利店库存和利润 | 支持经营类结局和便利店回忆。 |
| 图书馆正确匹配或知识增长 | 支持图书馆之星和旧地图回忆。 |
| 酒馆胜负与参与次数 | 支持酒馆传说和夜生活回声。 |
| 连续回家休息 | 支持安静稳定路线，不作为负面评价。 |

## 主结局集合

| 主结局 | 叙事方向 |
| --- | --- |
| 小镇明星经营者 | 玩家在多个地方都留下可靠身影。餐馆有人喊名字，便利店账本稳，图书馆借书卡多一行，酒馆也有人记得玩家。 |
| 冷酷赚钱机器 | 账本很好看，进出货和收益漂亮。这个结局不批判赚钱，只呈现更偏效率的生活画像。 |
| 受欢迎的普通居民 | 玩家没有制造大事件，但经常被人认出来。有人记得帮忙，有人记得买伞，也有人觉得玩家像住了很久。 |
| 图书馆之星 | 玩家把小镇旧事从书页和借书卡里翻出来。结局可以出现旧地图、泛黄纸页和管理员新写下的记录。 |
| 酒馆传说 | 玩家的夜晚留在棋盘和骰盅旁。常客未必把每局讲准确，但会记得玩家坐过的位置和输赢后的表情。 |
| 平凡小镇新人 | 玩家认真过了几天，也错过一些事。小镇没有被改变很多，但地图上多了一段被走过的路线。 |
| 经营失败者 | 十天不太顺，钱可能紧，体力也常常不够。但结局不羞辱玩家，仍保留“下次可以从慢一点的早晨开始”的余地。 |

## 判定与文案职责分离

P3 正式接入时应把两类逻辑分开：

- 结局判定：根据集中配置、最终属性、库存、路线统计和酒馆战绩选出唯一主结局。
- 结局文案：根据主结局和少量可解释依据生成最终显示文本。

结局判定必须可测试；文案选择可以轻量，但不能影响判定。

当前 P3 实现由 raylib-free 的 `EndingRules` 返回结局枚举、成长路线和判定依据，再由 `StoryText` 选择对应生活叙事。优先级固定为经营失败、综合高成就、单项路线、普通回退；单项路线按相对各自门槛的归一化分数比较，完全并列时稳定采用赚钱、声望、知识、娱乐顺序。库存先按集中配置清算，清算后的金钱参与结局判定。具体比例和阈值仍属于 P4 试玩平衡项。

## 第十天评议会职责

评议会负责连接十日过程和主结局：

1. 展示第十天结束的收束气氛。
2. 引入餐馆、便利店、图书馆、酒馆和镇长的记忆物件。
3. 给主结局一个自然入口。
4. 避免把结局页写成纯数值面板。

P1 可将评议会开场合并到结局页顶部。P3 若需要更清楚的节奏，可以单独增加评议会页面或结局前卡片，但必须评估 UI 流程和存档阶段。
