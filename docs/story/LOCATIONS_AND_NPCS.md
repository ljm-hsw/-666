# 地点叙事与 NPC 规划

本文描述五个地点的叙事职责、关键词、NPC 轻量支线和后续接入边界。MVP 不实现复杂 NPC 系统；本文用于 P2/P3 地点开发和 P4 内容扩充时保持方向一致。

## 地点叙事定位

| 地点 | 关键词 | 剧情功能 |
| --- | --- | --- |
| 餐馆 | 烟火气、忙碌、服务、热汤、熟客、老板嘴硬心软 | 代表小镇的日常热闹。玩家在这里感受到工作节奏、手忙脚乱和被熟客记住的过程。 |
| 便利店 | 天气、雨伞、汽水、便当、库存、价格、账本 | 代表小镇居民每天真实需要的东西。经营判断不只影响收益，也影响雨天有没有伞、夜里有没有饭团。 |
| 图书馆 | 旧地图、借书卡、泛黄书页、考试的孩子、管理员 | 代表安静、倾听和小镇过去。玩家在这里接触旧集市和居民记忆碎片。 |
| 酒馆 | 棋盘、骰子、输赢、常客、笑声、夜晚不孤单 | 代表夜晚的人情和一点风险。输赢重要，但更重要的是玩家是否在桌边留下位置。 |
| 家 | 窗灯、热水、疲惫、日记、明天、自我照顾 | 家不是跳过玩法，而是另一种有效选择。它让玩家把小镇生活过得稳定一点。 |

## A 级 NPC 三段式

| NPC | 叙事职责 | 初识 | 熟悉 | 回收 |
| --- | --- | --- | --- | --- |
| 镇长 | 主线入口和第十天评议。 | 递地图和钥匙。 | 偶尔在日初提示里出现。 | 主持评议会。 |
| 餐馆老板 | 烟火气和服务压力。 | 交代围裙和点单。 | 少说客套话。 | 在结局中提到玩家的服务片段。 |
| 便利店店主 | 经营判断和居民日常。 | 介绍货架和账本。 | 讨论天气与库存。 | 提到玩家让哪些东西没有断货。 |
| 图书馆管理员 | 记忆和倾听。 | 介绍分类。 | 交出旧地图或借书卡。 | 把玩家写进新借书记录。 |
| 酒馆老板或常客 | 夜晚、人情和输赢。 | 提醒规矩。 | 给玩家留座。 | 把玩家的棋局或骰局讲给别人听。 |
| 家 / 主角自我独白 | 自我照顾和日常节奏。 | 整理房间。 | 日记出现固定句式。 | 结局里承认休息也是生活路线。 |

## B 级触发碎片

| 触发示例 | 可触发内容 |
| --- | --- |
| 餐馆次数 >= 3 | 老板把旧集市大汤锅的来历说漏嘴。 |
| 图书馆次数 >= 2 且知识较高 | 管理员让玩家看旧地图，地图边缘有集市摊位标记。 |
| 便利店利润较高 | 店主把账本翻给玩家看，说这几天雨伞和便当都没压货。 |
| 酒馆胜场较多 | 常客给玩家起一个不太正式的外号。 |
| 连续两天回家休息 | 房间日记出现稳定生活片段，而不是负面评价。 |
| 声望较高 | 路上有人主动打招呼，日初提示里出现熟面孔。 |
| 金钱高但声望低 | 账本数字漂亮，但结局回忆里出现几次没接住的招呼。 |

## 接入边界

- P2 地点开发只需要提供玩法统计和一条行动摘要，不需要 NPC 状态机。
- P3 结局开发可以读取路线统计生成回收文本。
- P4 才允许扩写 NPC 三段式和旧集市碎片。
- 如果需要保存 NPC 进度，必须先评估存档格式变更，并补充存档向后兼容测试。
