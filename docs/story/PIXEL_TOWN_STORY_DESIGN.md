## 1. 剧情总基调

《像素小镇：十日经营计划》的剧情不是把玩家推上一个必须成功的位置。玩家只是暂时住进一座小镇，试着在十天里认识几条街、几扇门、几张熟面孔。有人会记得玩家端过的热汤，有人会记得便利店账本上的价格，也有人只记得某个雨夜酒馆里多坐了一个人。

整体气质应当放松、日常、温暖、不评判。游戏可以有经营压力、体力消耗和金钱波动，但这些数值不应把故事推向单一的胜负叙事。十天结束后，小镇给出的不是审判，而是一种回应：这段日子里，你像怎样的人一样生活过。

> 核心叙事问题<br>十天之后，小镇会怎样记住你？

后续所有开场、提示、地点文案、NPC 支线和结局文本，都应围绕这个问题工作。玩家不需要成为最强经营者，也不需要承担拯救小镇的责任。小镇只是打开门，让玩家进来住一阵子。

## 2. 世界观底盘

### 2.1 像素小镇现状

像素小镇曾经热闹过。街口旧海报还贴着夏日集市的日期，边角被风吹得卷起；餐馆后厨每天还开火，只是午饭高峰短了些；便利店夜里仍亮着灯，货架上有汽水、饭团和折得整齐的雨伞；图书馆的借书卡盒子没有扔，里面还夹着很多年前的名字；酒馆晚上会有人下棋，笑声不大，但能从窗缝里漏出来。

现在的小镇有些冷清，但并不荒凉。它更像一个等待被重新走熟的地方。玩家来到这里，不是为了修复所有问题，而是把十天过成自己的样子。

### 2.2 小镇十日共创计划

镇长提出一个很轻的邀请：以新居民的身份住进小镇十天。小镇会提供临时住所和一张地图，玩家可以在白天去餐馆、便利店或图书馆帮忙，晚上选择回家休息，或去酒馆玩一局棋、一把骰子。

“共创计划”不应写成宏大的建设项目。它更像镇长夹在文件夹里的一张纸，标题打得很正式，内容却很朴素：希望有人重新走走这些地方，看看镇上还有什么值得留下。

### 2.3 第十天评议会

第十天结束后，小镇会有一次简单评议。地点负责人、常客或镇长会把玩家十天里的选择拼在一起：哪几天去了餐馆，账本上留下什么数字，借书卡上多了什么记录，酒馆棋盘旁有没有人提起玩家。

评议会不是成绩单，也不是羞辱失败的场合。它只是把十天收束成一个生活画像，让玩家看到自己的选择如何被小镇记住。

### 2.4 旧夏日集市暗线

小镇曾经有一个夏日集市，后来停办了。这个暗线不进入 MVP 的强制完成目标，但可以作为后续剧情拓展的共同线索。

餐馆保留着集市时用过的大汤锅，老板嘴上说太占地方，却一直没扔。

便利店仓库里还有旧集市的价签和几箱没用完的纸杯。

图书馆夹着一张旧地图，上面圈出过集市摊位。

酒馆墙上有一张褪色照片，里面的人围着棋桌笑。

暗线的作用是给小镇增加纵深，而不是要求玩家完成隐藏任务。玩家可以碰到这些碎片，也可以只是过完自己的十天。

## 3. 叙事核心机制：选择 → 回声

剧情串联玩法的核心机制是“选择 → 回声”。玩家每天选择地点，不只是获得金钱、体力、声望、知识或心情变化，也会在后续文本里留下回声。回声可以很短，但要具体。

| 玩家选择 | 可能的叙事回声 |
| --- | --- |
| 多次去餐馆 | 老板开始少说客套话，熟客会记得玩家端汤的速度，结算时出现热汤、围裙、擦桌子等细节。 |
| 多次去便利店 | 店主会提到雨伞、汽水和便当的销量，晚归的人开始相信玩家的补货判断。 |
| 多次去图书馆 | 管理员递来旧地图，借书卡和泛黄书页成为小镇记忆的入口。 |
| 多次去酒馆 | 常客给玩家让座，棋盘和骰子成为夜晚的共同语言。 |
| 多次回家休息 | 文本强调窗灯、热水、日记和第二天，不惩罚玩家的安静路线。 |

MVP 阶段可以先用静态或少条件文本表达回声，不需要立即实现复杂条件系统。后续若加入触发条件，也应以地点次数、路线倾向和关键属性为主，不引入好感度或任务链。

## 4. 地点叙事定位

| 地点 | 关键词 | 剧情功能 |
| --- | --- | --- |
| 餐馆 | 烟火气、忙碌、服务、热汤、熟客、老板嘴硬心软 | 代表小镇的日常热闹。玩家在这里感受到工作节奏、手忙脚乱和被熟客记住的过程。 |
| 便利店 | 天气、雨伞、汽水、便当、库存、价格、账本 | 代表小镇居民每天真实需要的东西。经营判断不只影响收益，也影响雨天有没有伞、夜里有没有饭团。 |
| 图书馆 | 旧地图、借书卡、泛黄书页、考试的孩子、管理员 | 代表安静、倾听和小镇过去。玩家在这里接触到旧集市和居民记忆的碎片。 |
| 酒馆 | 棋盘、骰子、输赢、常客、笑声、夜晚不孤单 | 代表夜晚的人情和一点风险。输赢重要，但更重要的是玩家是否在桌边留下了位置。 |
| 家 | 窗灯、热水、疲惫、日记、明天、自我照顾 | 家不是跳过玩法，而是另一种有效选择。它让玩家把小镇生活过得稳定一点。 |

## 5. 剧情优先级层级

| 层级 | 目标 | 内容 | 实现压力 |
| --- | --- | --- | --- |
| S 级：MVP 必须实现 | 用最少文本让游戏不再像功能拼盘。 | 开场背景、十天每日提示、地点行动结果摘要、每日总结、第十天评议会、主结局文本。 | 不需要复杂 NPC 系统，不需要条件触发系统。短、自然、有效即可。 |
| A 级：强烈建议后续实现 | 让地点有“人”。 | 镇长、餐馆老板、便利店店主、图书馆管理员、酒馆老板或常客、主角自我独白。每个 NPC 使用初识、熟悉、回收三段式。 | 需要少量状态记录，但仍不需要好感度和对话树。 |
| B 级：后期拓展 | 剧情可以触发，也可以错过。 | 条件触发事件、NPC 支线碎片、旧夏日集市暗线、结局回忆片段。 | 需要根据地点次数、属性或路线倾向触发。 |
| C 级：远期设计预留 | 只记录方向，不进入当前开发压力。 | 复杂 NPC 支线、更强现实映射人物、多 NPC 交叉事件、隐藏结局、完整夏日集市线。 | 当前不实现，只作为未来扩展参考。 |

### 5.1 S 级：MVP 必须实现

S 级文本承担最基础的串联工作。它不需要让玩家读很多字，但要让玩家知道自己不是在反复进入功能模块，而是在过一段小镇日子。

开场背景：镇长递地图，玩家获得十天临时住所。

十天每日提示：每天一句天气、街景或小镇变化。

地点行动结果摘要：把数值变化包装进具体动作。

每日总结：用一两句收束当天选择。

第十天评议会：把十天选择汇总为结局前的生活回看。

主结局文本：表达小镇如何记住玩家。

### 5.2 A 级：强烈建议后续实现

A 级的目标是让地点有“人”，但仍然保持轻量。每个核心 NPC 不需要长篇剧情，只要在玩家多次接触后有一点变化。

| NPC | 叙事职责 | 三段式支线 |
| --- | --- | --- |
| 镇长 | 主线入口和第十天评议。 | 初识：递地图和钥匙。熟悉：偶尔在日初提示里出现。回收：主持评议会。 |
| 餐馆老板 | 烟火气和服务压力。 | 初识：交代围裙和点单。熟悉：少说客套话。回收：在结局中提到玩家的服务片段。 |
| 便利店店主 | 经营判断和居民日常。 | 初识：介绍货架和账本。熟悉：讨论天气与库存。回收：提到玩家让哪些东西没有断货。 |
| 图书馆管理员 | 记忆和倾听。 | 初识：介绍分类。熟悉：交出旧地图或借书卡。回收：把玩家写进新借书记录。 |
| 酒馆老板或常客 | 夜晚、人情和输赢。 | 初识：提醒规矩。熟悉：给玩家留座。回收：把玩家的棋局或骰局讲给别人听。 |
| 家 / 主角自我独白 | 自我照顾和日常节奏。 | 初识：整理房间。熟悉：日记开始出现固定句式。回收：结局里承认休息也是生活路线。 |

### 5.3 B 级：后期拓展

B 级剧情可以触发，也可以错过。它们适合在核心闭环稳定后加入，帮助重复游玩出现不同文本。

| 触发示例 | 可触发内容 |
| --- | --- |
| 餐馆次数 >= 3 | 老板把旧集市大汤锅的来历说漏嘴。 |
| 图书馆次数 >= 2 且知识较高 | 管理员让玩家看旧地图，地图边缘有集市摊位标记。 |
| 便利店利润较高 | 店主把账本翻给玩家看，说这几天雨伞和便当都没压货。 |
| 酒馆胜场较多 | 常客给玩家起一个不太正式的外号。 |
| 连续两天回家休息 | 房间日记出现稳定生活的片段，而不是负面评价。 |
| 声望较高 | 路上有人主动打招呼，日初提示里出现熟面孔。 |
| 金钱高但声望低 | 账本数字漂亮，但结局回忆里出现几次没接住的招呼。 |

### 5.4 C 级：远期设计预留

C 级只记录方向，不进入当前开发压力。它可以包括更复杂的 NPC 支线、多 NPC 交叉事件、隐藏结局和完整夏日集市线。若未来扩展，也应保持本项目的日常气质，避免突然变成任务驱动的大型剧情游戏。

## 6. 结局系统设计

结局不是好坏评价，而是生活路线画像。结局页应让玩家看到自己十天里的选择如何被小镇记住，而不是告诉玩家做得对或错。

主结局标题：保留现有七个方向，并允许后续扩展。

最终状态：展示金钱、体力、声望、知识、心情和酒馆战绩。

成长路线摘要：说明玩家十天里更常选择的生活方式。

小镇回忆片段：用一两件具体物件或动作收束结局。

| 主结局 | 叙事方向 |
| --- | --- |
| 小镇明星经营者 | 玩家在多个地方都留下了可靠的身影。餐馆有人喊名字，便利店账本稳，图书馆借书卡多了一行，酒馆里也有人记得玩家。 |
| 冷酷赚钱机器 | 账本很好看，进出货和收益都漂亮。只是有些招呼来得太轻，玩家可能没来得及接住。这个结局不批判赚钱，只呈现一种更偏效率的生活画像。 |
| 受欢迎的普通居民 | 玩家没有制造大事件，但经常被人认出来。有人记得玩家帮过忙，有人记得玩家买过伞，也有人只是觉得玩家像住在这里很久了。 |
| 图书馆之星 | 玩家把小镇的旧事从书页和借书卡里翻出来。结局可以出现旧地图、泛黄纸页和管理员新写下的记录。 |
| 酒馆传说 | 玩家的夜晚留在棋盘和骰盅旁。常客未必把每一局讲准确，但会记得玩家坐过的位置和输赢后的表情。 |
| 平凡小镇新人 | 玩家像大多数来过的人一样，认真过了几天，也错过了一些事。小镇没有被改变很多，但地图上多了一段被走过的路线。 |
| 经营失败者 | 这十天不太顺，钱可能紧，体力也常常不够。但结局不羞辱玩家。镇长仍会把地图收好，说下次可以从慢一点的早晨开始。 |

现有七种主结局可以作为 MVP 主结局集合，但不应被写成永远唯一的可能。后续可以添加回忆片段、支线尾声或隐藏结局，只要不破坏“每局一个主结局”的核心规则。

## 7. 文案风格规范

### 7.1 推荐

用具体物件和生活动作表达情绪，例如地图、围裙、雨伞、借书卡、棋子、窗灯、账本、旧海报。

短句优先。每段文字尽量只服务一个情绪或信息点。

NPC 说话要像普通人，不像系统旁白。

少解释，多呈现。不要替玩家总结过多。

结果文本先写场景，再写数值。数值是反馈，不是全部。

### 7.2 避免

宏大口号和宣传语。

直白说教，例如“你应该”“生活的意义是”。

过度煽情，尤其是用很重的词解释很轻的日常。

把玩家路线简单分成好坏。

用标签化方式写 NPC。

### 7.3 推荐写法 / 避免写法

| 场景 | 避免写法 | 推荐写法 |
| --- | --- | --- |
| 开场 | 你不需要拯救这里，也不需要证明自己。 | 镇长把地图递给你，纸角有点卷。“先住十天吧。小镇不大，走几遍就熟了。” |
| 餐馆 | 你通过努力让餐馆重新充满希望。 | 午饭刚过，老板把围裙往柜台上一搭。“今天汤没洒，已经很不错了。” |
| 便利店 | 你满足了居民的核心生活需求。 | 雨伞卖得比汽水快。店主在账本边角画了个小勾，说晚上应该还能再来两个人。 |
| 图书馆 | 你发现了小镇过去的意义。 | 借书卡盒子有点卡。管理员敲了两下，掉出一张旧集市地图。 |
| 酒馆 | 你的胜利让大家尊敬你。 | 骰盅停下时，旁边的人先笑了。“这局算你运气好。下一把别走。” |
| 失败结局 | 你经营失败，没有完成小镇期待。 | 账本最后一页有几处空白。镇长没有多说，只把地图折好，放回你手边。 |

## 8. MVP 可直接使用的文本草案

### 8.1 开场文本

> 开场文本草案<br>车在小镇入口停下时，路边的旧海报被风掀起一角。镇长拿着钥匙和地图等在站牌旁，地图折痕很深，像被很多人借过。“先住十天吧。”他说，“餐馆、便利店、图书馆、酒馆都在图上。想去哪里，就从哪里开始。”

### 8.2 十天每日提示

| 游戏日 | 每日提示 |
| --- | --- |
| 第 1 天 | 镇长把地图摊在桌上，铅笔圈出几个地点。今天先走一遍也没关系。 |
| 第 2 天 | 餐馆门口的菜单换了新纸，墨还没干透。街上有人比昨天更早出门。 |
| 第 3 天 | 天气晴。便利店门口摆出汽水箱，箱角压着一张旧集市传单。 |
| 第 4 天 | 图书馆窗台晒着借书卡。管理员说，今天适合把书页翻慢一点。 |
| 第 5 天 | 傍晚可能下雨。店主把雨伞往门边挪了挪，又看了一眼账本。 |
| 第 6 天 | 酒馆昨晚的棋盘还没收。有人把一枚黑子落在了桌缝旁。 |
| 第 7 天 | 餐馆门口多摆了一张桌子。老板说只是天气好，适合坐外面。 |
| 第 8 天 | 旧海报被重新贴平了。图书馆那边传来扫地的声音。 |
| 第 9 天 | 小镇路不长，但你已经能记住哪盏灯最晚熄。 |
| 第 10 天 | 镇长说晚上大家会简单聊聊这十天。桌上已经放好了账本和几张便签。 |

### 8.3 地点短结果摘要

| 地点 | 可用摘要 |
| --- | --- |
| 餐馆 | 你把最后一碗汤端到靠窗的位置，老板只点了点头。午饭高峰过去，桌面还热着。围裙沾了点汤汁，但熟客说今天上菜挺快。 |
| 便利店 | 雨伞少了三把，饭团也卖空一排。店主把零钱理进抽屉，账本边角多了一个小勾。汽水卖得慢些，不过夜里来买咖啡的人比预想多。 |
| 图书馆 | 你帮孩子找到了考试要用的书，借书卡又多盖了一个章。管理员把旧地图递给你，说可以先看看，不急着还。书页翻动的声音很轻，窗外的街也慢下来。 |
| 酒馆 | 棋子落下去的声音不大，旁边的人却都看了过来。骰盅打开时，有人笑着拍了拍桌子。你输了这一局，但椅子还给你留着。 |
| 家 | 你烧了热水，把今天的事写进日记。窗灯亮到很晚，街上的声音慢慢退下去。你没有去别的地方，但明天醒来时，体力会好一些。 |

### 8.4 第十天评议会开场

> 评议会开场草案<br>第十天晚上，镇长把几张便签摊在桌上。餐馆老板带来一张油点很小的菜单，便利店店主拿着账本，图书馆管理员夹着几张借书卡，酒馆那边有人拎来一副棋子。镇长清了清嗓子。“十天不长，”他说，“但也够大家记住一些事了。”

### 8.5 七个主结局初版文案方向

## 9. 与现有项目约束的关系

本剧情设计文档服务于现有 PRD、CONTEXT 和 DESIGN，不改变其中的核心领域约束。它提供文案和叙事方向，不要求新增复杂剧情系统。

不修改 C++ 逻辑代码。

不新增复杂好感度、对话树、任务链或开放世界系统。

不改变十日周期、地点模块、统一行动结果、单主结局等核心规则。

S 级内容可先用静态或少条件文本落地。

A/B/C 级用于后续迭代排期，不制造当前 MVP 开发压力。

后续如果需要进入实现阶段，建议先把 S 级文本整理成数据文件或资源表，再由核心系统在开场、每日提示、地点结果、每日总结和结局页读取。NPC 支线与旧夏日集市暗线应等核心闭环稳定后再排期。
