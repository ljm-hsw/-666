# 餐馆视觉布局与资源槽位规格

本文定义 Issue 21 的餐馆 960×540 页面规格。它约束 P2 程序绘制 UI、后续 imagegen 候选素材和 P4 最终美术替换边界。

## 目标

餐馆是白天反应型工作场景。玩家必须在一个画面内看清：

- 当前订单是什么。
- 应该选择哪个食物。
- 还剩多少时间。
- 刚才操作是正确、错单还是超时。
- 最后结算带来了哪些属性变化。

最终素材可以后续替换，但这些信息结构不能丢失。

## 960×540 布局

| 区域 | 坐标建议 | 用途 | 替换边界 |
| --- | --- | --- | --- |
| 全局状态栏 | `0,0,960,84` | 天数、阶段、金钱、体力、声望、知识、心情、静音状态 | 由全局 UI 维护，餐馆背景不能遮挡 |
| 餐馆背景安全区 | `0,84,960,456` | 餐馆室内背景、吧台、餐桌、厨房窗口 | 可由 imagegen 960×540 背景替换；不能包含文字 |
| 主交互面板 | `34,96,592,250` | 说明卡、订单票据、反馈、结算信息 | 程序绘制面板；P4 可替换边框/材质 |
| 说明卡正文 | `62,160,520,120` | 目标、操作、收益、风险、结束方式 | 文本必须由程序字体绘制 |
| 订单票据 | `58,134,240,96` | 当前第几单、菜品、剩余时间 | 可用票据装饰图，但文字由程序绘制 |

## 背景资源要求

餐馆室内背景是候选 imagegen 资产，不阻塞 P2 逻辑。生成要求：

- 尺寸：`960×540`。
- 风格：温暖、清晰、轻量像素风，硬边、低噪点。
- 内容：吧台、餐桌、厨房或上菜窗口、墙面装饰、地板。
- 安全区：上方 `0-84` 不放关键视觉；左中区域适合叠加主交互面板。
- 禁止：中文、英文、数字、水印、菜单文字、可读招牌。
- 运行策略：背景缺失时继续使用程序绘制餐馆场景，不影响测试和玩法。

建议路径：

- 候选背景：`assets/textures/imagegen_backgrounds/restaurant_interior_candidate.png`
- 最终背景：由 P4 人工验收后再确认命名和 CREDITS 记录。

## 食物图标槽位

五个菜品必须在小尺寸下能区分：

| 菜品 | 识别特征 | 当前 fallback | 后续图标要求 |
| --- | --- | --- | --- |
| 炒饭 | 米饭盘、绿色/红色配料点 | 程序绘制圆盘和配料点 | 暖黄色米饭，不含文字 |
| 面条 | 碗或盘中线条状面条 | 程序绘制浅色线条 | 线条轮廓明显 |
| 汤 | 红/橙汤面、小绿叶 | 程序绘制红汤面 | 与炒饭色块明显不同 |
| 饺子 | 三个浅色半圆/团 | 程序绘制三个小圆 | 小尺寸仍像饺子 |
| 沙拉 | 绿色叶片和红色点缀 | 程序绘制绿色圆块 | 绿色主体，不能像汤 |

建议 spritesheet：

- 尺寸：`5 × 48×48` 或 `5 × 64×64`。
- 背景：透明 PNG 或可处理的纯色 chroma-key。
- 文本：禁止烘焙数字、菜名或标签。
- 运行策略：图标缺失时使用当前程序图标和文字按钮。

## 时间表现

等待时间必须同时有：

1. 数字文本，例如 `剩余 3 秒`。
2. 条形进度，宽度随剩余时间减少。
3. 低时间警示，剩余 `<= 1 秒` 或比例 `<= 34%` 时使用红色。

不得只靠文字秒数表达时间压力。

## 反馈表现

餐馆反馈状态分为：

| 状态 | 视觉要求 | 规则来源 |
| --- | --- | --- |
| 正确 | 绿色或积极反馈，短句说明顾客认可 | `RestaurantFeedback::correct` |
| 错单 | 红色或警示反馈，说明会影响表现 | `RestaurantFeedback::wrong` |
| 超时 | 红色警示，说明顾客离开 | `RestaurantFeedback::timeout` |
| 等待 | 中性说明，不算结果 | `RestaurantFeedback::none` |

反馈页不直接修改玩家状态，只提示地点内部统计已经变化。最终属性变化仍在完成结算时统一应用。

## 说明卡内容

说明卡必须覆盖五类信息：

- 目标：按订单上菜。
- 操作：`1-5` 和鼠标点击。
- 收益：金钱、声望、完美服务心情。
- 风险：错单、超时、等待归零。
- 结束：服务完结算；主动放弃消耗白天且无收益。

说明卡文字由 `restaurant_ui_model` 提供，绘制层只负责排版。

## 结算页信息

完成页至少显示：

- 正确、错单、超时。
- 正确率。
- 金钱、体力、声望、心情变化。
- “点击完成结算后进入夜晚选择”的操作提示。

结算页可以显示数值，但长剧情摘要不在绘制代码中硬编码，后续通过 `StoryText` 或地点文本 adapter 收口。

## 资产记录要求

任何餐馆 imagegen 或第三方资产进入仓库前，必须记录：

- 用途：背景、食物图标、票据、时间条或反馈标记。
- 状态：candidate / approved / final。
- 来源：imagegen prompt、Kenney、手工绘制或其他。
- SHA-256。
- 许可或生成说明。
- 人工验收截图或说明。

候选资产不能默认为最终交付资产。
