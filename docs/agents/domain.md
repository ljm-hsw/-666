# Domain Docs

本仓库采用 single-context 领域文档布局。工程技能在探索代码或生成 issue 前，按本文件定位领域术语与架构决策。

## 开始探索前读取

- 根目录 `CONTEXT.md`：项目领域概念、统一术语及明确不使用的近义表达。
- `docs/adr/`：与当前工作范围有关的架构决策记录。

如果这些文件尚不存在，继续当前任务，不将缺失本身视为阻塞，也不为填充目录而创建空文档。领域概念或架构决策实际形成后再写入。

## 文件结构

```text
/
├── CONTEXT.md
├── docs/
│   └── adr/
│       ├── 0001-<decision>.md
│       └── ...
└── ...
```

本仓库不使用 `CONTEXT-MAP.md`。如果未来演变为包含多个独立业务上下文的 monorepo，应先更新本文件，再迁移为 multi-context 布局。

## 使用统一领域语言

issue 标题、重构建议、诊断假设、测试名称和用户文档应使用 `CONTEXT.md` 定义的术语。需要的概念尚未定义时，先判断是在发明不必要的同义词，还是发现了真实领域缺口；后者应进入领域文档讨论。

## 报告 ADR 冲突

如果建议或实现与现有 ADR 冲突，必须明确指出冲突的 ADR、冲突内容和重新讨论的理由，不得静默覆盖既有决策。
